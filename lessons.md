# lessons

- 界面位置问题必须运行新构建产物并截图验证，不能只靠静态 diff 判断完成。
- 验证前必须确认当前运行进程路径，避免用户看到的还是旧 exe。
- 任务栏文字验证必须覆盖用户当前字号；固定窗口尺寸会在大字号下裁切第二行和金额。
- Windows 原生设置面板里中文标签不要依赖窄静态控件，优先自绘或留足宽度，否则 DPI 下容易截字。
- 设置页不仅要看标签是否完整，还要看输入框内容是否被微调箭头挤压；截图必须覆盖所有可见控件。
- DPI 自适应不能简单把整个设置面板按显示缩放倍数放大，否则高 DPI 屏上弹窗会过大；优先统一系统字体和控件尺寸。
- 类似豆包/ccswitch 的右键菜单字号应优先使用 Windows 系统菜单字体 `NONCLIENTMETRICS.lfMenuFont`，不要手动猜字号。
- 数字微调控件在轻量弹窗里容易挤压输入框，真实截图不稳定时优先去掉微调控件，保留直接输入。
- 复选框不能只画浅灰细框，选中状态必须有明显色块和勾，截图里要能一眼看出来。
- 设置面板表格式布局要先定义统一行高、控件高度和右列起点，再按这些常量放置文字、输入框、色块和复选框。
